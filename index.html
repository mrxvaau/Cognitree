How it works

The Cognitree Pipeline

Five stages that transform a forgetful LLM into a precise, state-aware coding agent.

01
🧠
LLM Completes a Task
Claude Code, Qwen Code, or Gemini CLI produces output — code, a decision, a fix, an explanation.
02
⚙️
Memory Extractor Parses Output
Cognitree scans the output for decisions, errors, facts, and state changes — tagging each for its memory layer.
extract({ decisions: [...], // WHY something was done errors: [...], // What failed facts: [...], // What exists in the project state: {...} // Current task state })
03
🗄️
Writes to Memory Store
Extracted data is written to a structured local memory store — layered by type, tagged by project, timestamped.
04
🎯
Context Injector Fires Before Next Prompt
Before the next session begins, Cognitree retrieves relevant memories and injects a structured block into the system prompt.
[COGNITREE MEMORY — VoidRoom] Stack: JavaScript + MongoDB Last: Fixed auth route → /api/auth.js ERROR LOG: mongoose.findByIdAndDelete() → use findOneAndDelete() — hallucinated Decision: JWT over sessions (stateless req) [/COGNITREE]
05
🚀
LLM Works with Perfect Precision
The model now knows exactly what it did, what failed, what exists, and why decisions were made — zero re-derivation, zero wasted tokens.
Architecture

Four Memory Layers

Each layer stores a different type of knowledge, queried and injected with surgical precision.

Layer 1 — Working
Session Memory
Short-lived, high-priority context. What is happening right now in this session.
Current file being edited
Last 5 actions taken
Active variables & scope
Layer 2 — Episodic
Task Memory
Decisions, reasoning chains, and outcomes from completed tasks. The "why" behind every choice.
Why approach X was chosen
What was tried and reverted
Architecture decisions + dates
Layer 3 — Semantic
Project Memory
Structured knowledge about the codebase — what exists, what connects to what, how it's organized.
Stack, deps, file structure
Module roles & API routes
Coding style & conventions
Layer 4 — Error Log
Mistake Memory
The most powerful layer. Hallucinations, wrong paths, failed patterns — logged to prevent recurrence forever.
Hallucinated functions / APIs
Wrong file paths tried
Patterns that caused bugs
Compatibility

Memory injected via CLAUDE.md project file and system prompt prepending per session.
🟣
Qwen Code

Gemini CLI

GEMINI.md memory file plus persistent session state via Cognitree's local store API.
